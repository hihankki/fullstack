# ЛР1 — RBAC (Roles + Permissions) для MVP (FastAPI + React)

## Роли
- **guest** — неавторизованный пользователь
- **user** — авторизованный пользователь
- **admin** — администратор

## Permissions
- `reviews:read` — просмотр отзывов
- `reviews:create` — создание отзыва
- `reviews:update_own` — редактирование своего отзыва
- `reviews:delete_own` — удаление своего отзыва
- `reviews:update_any` — редактирование любого отзыва
- `reviews:delete_any` — удаление любого отзыва
- `users:manage_roles` — управление ролями пользователей

## Матрица (роль → действия)

| Action / Permission         | guest | user | admin |
|----------------------------|:-----:|:----:|:-----:|
| Читать отзывы              |  ✅   |  ✅  |  ✅   |
| Создавать отзыв            |  ❌   |  ✅  |  ✅   |
| Редактировать свой отзыв   |  ❌   |  ✅  |  ✅   |
| Удалять свой отзыв         |  ❌   |  ✅  |  ✅   |
| Редактировать любой отзыв  |  ❌   |  ❌  |  ✅   |
| Удалять любой отзыв        |  ❌   |  ❌  |  ✅   |
| Менять роли пользователей  |  ❌   |  ❌  |  ✅   |

## Доп. ограничения
- **user** может менять/удалять **только свои** отзывы (ownership check)
- отказ в доступе:
  - без токена → `401 Unauthorized`
  - без прав → `403 Forbidden`

## Реализация
### Backend (FastAPI)
- `backend/rbac.py` — роли, permissions, матрица и guard
- `PATCH /api/admin/users/{username}/role` — смена роли (только admin)

### Frontend (React)
- скрытие/показ действий по роли (админка видна только admin)
- защита маршрутов:
  - `/my`, `/create` → только авторизованные
  - `/admin/users` → только admin
